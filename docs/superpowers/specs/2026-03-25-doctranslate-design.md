# DocTranslate 设计文档

## 项目概述

**项目名称**：DocTranslate

**核心价值**：本地文档翻译 + 智能问答工具

**目标用户**：需要将中文文档转为双语 PDF，并对文档内容进行问答的用户

### 技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端 | Vue 3 + TypeScript + Element Plus |
| 后端 | Tauri 2.0 (Rust) |
| 向量存储 | LanceDB |
| 元数据存储 | SQLite |
| AI 服务 | DeepSeek / Moonshot / 智谱 / 百炼 API |

### 核心模块

1. **设置模块** — 配置模型提供商（API Key、Base URL、模型选择）
2. **文档管理** — 上传、列表、删除文档
3. **toPdf 任务** — 创建翻译任务，异步处理，下载结果
4. **聊天问答** — 多文档选择，基于向量检索的回答

#### 文档上传流程
```
用户上传 → 存储本地文件 → 解析提取文本 → 向量化存入 LanceDB
```

#### 任务处理流程
```
创建任务 → 读取文档 → 并行翻译 → 生成双语 PDF → 任务完成通知
```

#### 聊天问答流程
```
选择文档 → 输入问题 → 向量检索相关段落 → 构建提示词 → 调用 AI → 返回答案
```

## 数据模型

### 核心实体定义

```typescript
// 模型提供商配置
interface ProviderConfig {
  id: string;           // deepseek | moonshot | zhipu | bailian
  name: string;         // 显示名称
  apiKey: string;       // API 密钥
  baseUrl: string;      // API 地址（有默认值）
  model: string;        // 选择的模型
  embeddingModel: string; // 嵌入模型
  isActive: boolean;    // 是否启用
}

// 文档
interface Document {
  id: string;
  name: string;
  type: 'word' | 'pdf';
  size: number;         // 字节数
  path: string;         // 本地存储路径
  createdAt: Date;
  vectorized: boolean;  // 是否已向量化
}

// 任务
interface Task {
  id: string;
  documentId: string;
  status: 'pending' | 'processing' | 'completed' | 'failed';
  progress: number;     // 0-100
  outputPath?: string;  // 完成后的输出路径
  error?: string;
  createdAt: Date;
  completedAt?: Date;
}

// 聊天会话
interface ChatSession {
  id: string;
  documentIds: string[]; // 选中的文档
  messages: ChatMessage[];
  createdAt: Date;
}

interface ChatMessage {
  id: string;
  role: 'user' | 'assistant';
  content: string;
  createdAt: Date;
}
```

### 存储位置

| 数据类型 | 存储方式 |

| 任务数据 | SQLite |
| 聊天记录 | SQLite |

## 模块详细设计

### 4.1 设置模块

**功能：**
- 配置模型提供商（DeepSeek、Moonshot、智谱、百炼）
- 输入 API Key、选择模型
- 测试连接有效性

**界面元素：**
- 提供商列表（卡片或下拉选择）
- 每个提供商的配置表单（API Key、可选 Base URL、模型下拉）
- "测试连接"按钮
- 保存配置按钮

**处理流程：**
1. 用户填写 API Key
2. 点击测试 → 调用后端验证接口
3. 显示成功/失败提示
4. 保存到 Tauri 配置文件

---

### 4.2 文档管理模块

**功能：**
- 上传文档（Word/PDF）
- 文档列表展示
- 删除文档
- 文档向量化状态显示

**界面（左侧常驻面板）：**
- 顶部：上传按钮
- 列表：文档名称、类型图标、大小、向量化状态
- 每行：删除按钮

**处理流程：**
```
用户上传 → 存储到本地数据目录 → 后台解析提取文本
→ 调用嵌入 API 获取向量 → 存入 LanceDB → 更新向量化状态
```

**向量化策略：**
- 上传后自动向量化
- 失败时显示"未向量化"，可手动重试

---

### 4.3 toPdf 任务模块

**功能：**
- 选择文档创建翻译任务
- 查看任务列表和状态
- 查看任务进度
- 下载/打开完成的 PDF
- 重试失败任务

**界面（右侧内容区）：**
- 任务列表：文档名、状态、进度条、创建时间
- 操作按钮：创建任务、下载、打开、删除、重试

**处理流程：**
1. 用户选择文档 → 点击"创建任务"
2. 后端创建任务，状态为 pending
3. 任务调度器取出任务，状态变为 processing
4. 解析文档 → 提取段落 → 并行翻译（4-6并发）
5. 翻译时维护术语表，保证一致性
6. 生成双语 PDF（段落对照格式）
7. 完成 → 状态 completed，记录输出路径

**并发策略：**

| 内容量 | 处理策略 |
|--------|----------|
| < 5000字 | 单线程同步处理 |
| 5000-20000字 | 多线程翻译（4-6并发），异步处理 |
| > 20000字 | 多线程 + 进度条，异步队列 |

**进度反馈：**
- 实时进度条（已翻译段落数/总段落数）
- 通过 Tauri 事件推送前端更新

**PDF 输出格式：**
- 段落对照：每段中文后紧跟英文翻译
- 复杂元素（图片、表格、公式）保留原位
- 仅翻译中文文字

**术语一致性：**
- 翻译过程中维护术语表
- 同一术语保持统一翻译

---

### 4.4 聊天模块

**功能：**
- 多文档选择
- 基于文档的问答
- 普通聊天（不选文档时）
- 聊天记录保留

**界面（右侧内容区）：**
- 顶部：文档选择器（多选下拉/标签）
- 中间：聊天消息列表
- 底部：输入框 + 发送按钮

**处理流程：**

**有选文档时：**
1. 向量检索相关段落（Top-K，5-10 段）
2. 构建提示词：系统提示 + 相关段落 + 用户问题
3. 调用 AI 生成回答
4. 如果检索无结果，回复"在所选文档中未找到相关信息"

**无选文档时：**
- 直接调用 AI 普通对话

**检索策略：**
- 使用嵌入向量进行相似度搜索
- 返回相似度最高的 K 个段落
- 相似度阈值过滤（低于阈值视为无结果）

## 错误处理

### API 相关错误

| 错误类型 | 处理方式 |
|----------|----------|
| API Key 无效 | 提示用户检查配置 |
| 余额不足 | 提示充值或更换提供商 |
| 限流 | 自动退避重试，显示"请求频繁，稍后重试" |
| 网络超时 | 重试 3 次，失败后标记任务失败 |

### 文件相关错误

| 错误类型 | 处理方式 |
|----------|----------|
| 文件损坏/格式不支持 | 提示"文件解析失败" |
| 文件过大 | 提示建议，但仍允许处理 |
| 磁盘空间不足 | 提示清理空间 |

### 向量化失败处理

- 文档列表显示"未向量化"状态
- 提供手动重新向量化的入口
- 聊天时检测文档未向量化，提示用户先向量化

### 边界情况

| 情况 | 处理方式 |
|------|----------|
| 空文档 | 提示"文档无内容" |
| 纯图片文档 | 提示"无可翻译文本" |
| 超长文档 | 分段处理，提示用户关注进度 |
